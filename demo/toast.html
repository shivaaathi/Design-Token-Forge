Toast Explorer — Design Token Forge

~90 CSS Variables 4 Roles 6 Positions Auto-Dismiss 7 Axes

Hero Preview

Live

Click buttons to fire live toasts. They stack, auto-dismiss with progress bar, and can be manually closed.

Semantic Roles

Surface

Info, success, warning, danger. Each role colors the icon, accent bar, progress, and title.

Anatomy

Slots

Accent bar, icon, title, body, action button, close button, and progress bar — all optional slots.

Positions

Layout

Six fixed positions for the toast container. Container is a flex column that stacks toasts.

Actions

Interactive

Toasts can include an inline action button (e.g. "Undo") styled with the role's accent color.

Auto-Dismiss

Motion

Progress bar shrinks over --toast-auto-dismiss duration (5s default). Click the progress demo to start.

Accessibility

A11y

Toasts use role="status" for polite announcements. Danger uses role="alert" for assertive.

ARIA Roles

Info/success/warning = role="status". Danger = role="alert".

Keyboard

Close button is focusable. Action button is tabbable.

Forced Colors

Border + accent bar visible. Progress bar uses Highlight.

Use in Your Framework

Integration

Wrap this component in your framework. Props become data attributes — our CSS does the rest.
