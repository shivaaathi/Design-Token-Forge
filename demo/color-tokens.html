System Overview

Architecture

The Tokn 4-tier color system flows from raw palette values (T0 Primitives) through themed semantic roles (T1 Semantic) and surface contexts (T2 Surfaces) down to individual component tokens (T3 Component). Each tier adds meaning and context, ensuring consistent theming across light and dark modes.

T0 — Primitive Palettes

176 colors

8 handcrafted palettes, each with 22 perceptual steps from white → black. Step 500 is the key color. These raw values never appear in component CSS directly — they feed into T1 semantic roles.

T1 — Semantic Roles

216 tokens

6 semantic roles, each with 18 purpose-driven tokens divided into Content (4), Component (7), Container (5), and OnComponent Content (2) groups. Values change between light and dark themes while token names stay the same.

Light ↔ Dark Comparison

Theme

Side-by-side comparison of how the same semantic token resolves to different primitive steps depending on the active theme. Light uses darker steps for content, dark inverts.

T2 — Surface Contexts

224 tokens

8 surface contexts define background, content, and component colors for different UI regions. Brightness order: Bright > Base > Dim > Deep. Elevation order: Float > Over-container > Container > Base.
