How This Design System Works

This design system is built on a three-tier architecture that creates a single source of truth for all design decisions.

1

Tier 1: Primitives (No Context)

Raw design values with no semantic meaning. These are the atoms of the system — colors organized by hue and luminance, spacing values in pixels, typography sizes in relative units.

--brand-500: #7C3AED
--grey-100: #F0F0F0
--sp-16: 16px
--fs-24: 24px
2

Tier 2: Semantic Tokens (Named by Intent)

Semantic aliases that assign meaning to primitives. These tokens answer the question "what is this for?" They exist in light and dark variants, enabling seamless theme switching.

--bg-primary: var(--grey-0)
--text-primary: var(--grey-900)
--interactive-primary: var(--brand-500)
--border-primary: var(--grey-200)
3

Tier 3: Usage-Ready Components

Fully built components that apply semantic tokens in context. Buttons, cards, inputs, and all other UI elements are constructed from Tier 2 tokens. When you change a Tier 2 token, all Tier 3 components update automatically.

Button background: var(--interactive-primary)
Button hover: var(--interactive-hover)
Card padding: var(--sp-20)
Card title font: var(--fs-20), var(--fw-semi)

Why This Structure Matters

  • Single source of truth: Change a primitive blue → all brand-related tokens update → entire UI respects the change.
  • Semantic clarity: Designers and developers speak the same language ("use the interactive-primary color").
  • Theme flexibility: Light and dark modes are built in. Switch between them instantly across the entire system.
  • Scale instantly: Add a new primitive color once. It cascades through semantic tokens to all components automatically.
  • Product variants: Create different product token sets without rebuilding components.
